2. 1주차 주요 개념 키워드 알아보기
	-버전 관리(형상관리) : 프로젝트의 변경되는 정보들을 관리 하는 것으로 이를 통해 다른 개발자들과 협업 할 수 있다
	-git 초기화 : 로컬 repo에 있는 폴더를 git이 관리 할 수 있도록 초기화 하는 것
	-commit : 본인이 현재 작업 한 내용에 대해서 snapshot을 찍듯이 저장 하는 것
	-add(staging) : commit에 반영하는 것을 선택해서 할 수 있는 것 -> 이를 통해 추후 모듈별로 작업 내용을 commit
	-commit history : commit한 기록(날짜/메세지/작성자 등)
	-push/pull : push는 로컬에서 원격저장소로, pull은 원격에서 로컬 저장소로 반영 하는 것이다.
	-clone : 원격 저장소의 코드를 로컬 저장소로 초기에 세팅하는 것.
