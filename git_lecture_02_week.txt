1. 앞으로 git 프로젝트로 만들고 싶은 것
	- 회사에서 gitlab을 통해서 단순히 코드만 공유하는 기능으로 쓰고 정확한 협업 개념을 모르고 기계적으로 사용했었는데 이번 수업을 통해서 브랜치,이슈,머지 등에 대해서 알게 되어서 좀 더 협업하는데 큰 도움이 되었습니다.

2. 2주차 주요 개념 키워드 알아보기
	-Issue : 작업 할당 단위(내가 할 일/버그 등등)
	-Branch : 각자 할당 맡은 Issue를 작업하는 공간으로 각각의 branch는 서로 영향을 주지 않는다
	-Merge : 각자의 branch를 main branch에 병합하는 작업, 같은 파일을 병합시에는 merge conflict 조심하기
	-Branch 관리 : 규칙을 가지고 이름을 정해야 나중에 프로젝트가 커졌을때 관리가 용이해진다.
	-Tracking : 원격 repo와 로컬 repo를 tracking 한다는 것은 브랜치 단위로 서로 commit내역을 관리하는 것
